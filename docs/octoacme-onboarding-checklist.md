# OctoAcme — Team Member Onboarding Checklist

## Purpose
Help new team members quickly understand OctoAcme's project management processes and contribute effectively.

## Before Day 1
- [ ] Send project management docs overview
- [ ] Share current project charter and roadmap
- [ ] Add new member to team channels and project board access

## Week 1
- [ ] **Day 1:** Meet with manager; intro to team and current projects
- [ ] **Day 1-2:** Read [Project Management Overview](octoacme-project-management-overview.md)
- [ ] **Day 2:** Attend standup; observe project rhythm
- [ ] **Day 2-3:** Review relevant process docs based on role (see below)
- [ ] **Day 3:** Meet with PM and/or PdM; understand current project status
- [ ] **Day 5:** Attend weekly PM-PdM sync or delivery standup

## Role-Specific Onboarding

### New Developers
- [ ] Read [Execution & Tracking](octoacme-execution-and-tracking.md)
- [ ] Review current PR conventions and CI setup
- [ ] Clone repo; run local tests
- [ ] Submit first small PR with team guidance
- [ ] Attend code review session

### New Product Managers
- [ ] Read [Project Initiation](octoacme-project-initiation.md) and [Project Planning](octoacme-project-planning.md)
- [ ] Review 2-3 past Project One-pagers
- [ ] Review current project's backlog and roadmap
- [ ] Meet with current PdM on prioritization approach

### New Project Managers
- [ ] Read all process docs (start with [Overview](octoacme-project-management-overview.md))
- [ ] Review Risk Register template and past examples
- [ ] Shadow current PM in status update and escalation
- [ ] Attend retrospective to see improvement cycle

### New QA/Testing
- [ ] Read [Execution & Tracking](octoacme-execution-and-tracking.md)
- [ ] Review test plan and acceptance criteria from current project
- [ ] Understand CI/CD pipeline and test automation setup
- [ ] Run smoke test suite locally

## Week 2-4
- [ ] **Week 2:** Contribute to your first backlog item (with pairing/guidance)
- [ ] **Week 3:** Lead or co-lead a small activity in your area (e.g., estimate story, run test, draft docs)
- [ ] **Week 4:** Attend retrospective; observe improvement cycle

## 30-Day Check-in
- [ ] Async feedback from onboarding buddy
- [ ] Self-assessment: confidence level on key processes
- [ ] Manager one-on-one: areas for additional support
- [ ] Celebrate first month on team!
